# 6장. 반복의 마법 — for / while

지금까지 우리는
조건에 따라 **선택**을 했다.

그런데 이런 생각이 들기 시작한다.

> “같은 일을
> 여러 번 해야 하면 어떻게 하지?”

이 질문에 대한 답이 바로 **반복문**이다.
반복문은 **귀찮은 일을 대신해 주는 도구**다.

---

## 💬 GPT에게 이렇게 물어보자

> **“for문과 while문은 왜 필요한가요?”**

GPT는 보통 이렇게 설명한다.

* 같은 코드를 여러 번 실행할 때
* 규칙적으로 반복해야 할 때
* 실수를 줄이기 위해서

아주 간단히 말하면 이렇다.

> **반복문은
> ‘똑같은 일을 여러 번 시키는 방법’이다.**

---

## 🔁 반복을 눈으로 이해해 보자

예를 들어 이런 일을 시키고 싶다고 하자.

* “안녕하세요”를 10번 출력하기

가장 단순한 방법은
`printf`를 10번 쓰는 것이다.

하지만 반복문을 쓰면
**한 줄의 규칙**으로 끝난다.

---

## ▶ for문: 횟수가 정해진 반복

아래 코드를 실행해 보자.

```c
#include <stdio.h>

int main(void)
{
    int i;

    for (i = 1; i <= 10; i++)
    {
        printf("안녕하세요 %d\n", i);
    }

    return 0;
}
```

실행 결과는 다음과 같다.

```
안녕하세요 1
안녕하세요 2
안녕하세요 3
...
안녕하세요 10
```

여기서 핵심은 이것이다.

> **for문은
> “몇 번 반복할지”가 보인다.**

---

## 🔍 for문의 구조를 질문으로 바꿔 보자

```c
for (i = 1; i <= 10; i++)
```

이 줄은 사실
세 개의 질문으로 이루어져 있다.

1. 처음에 i는 얼마인가? → `i = 1`
2. 언제까지 반복할까? → `i <= 10`
3. 한 번 끝나면 무엇을 할까? → `i++`

이 세 가지가 합쳐져
**반복의 규칙**이 된다.

---

## ▶ while문: 조건이 중요할 때

이번에는 `while`문을 사용해 보자.

```c
#include <stdio.h>

int main(void)
{
    int i = 1;

    while (i <= 10)
    {
        printf("Hello %d\n", i);
        i++;
    }

    return 0;
}
```

실행 결과는 for문과 같다.

---

## 🤔 for와 while, 언제 쓸까?

이 질문도 GPT에게 던져 보자.

---

## 💬 GPT에게 이렇게 물어보자

> **“for문과 while문은 언제 각각 쓰나요?”**

GPT의 답을 한 줄로 정리하면 이렇다.

* **for문**: 반복 횟수가 분명할 때
* **while문**: 언제 끝날지 조건이 중요할 때

---

## ⚠️ 가장 많이 겪는 문제: 무한 반복

아래 코드를 실행하면 어떻게 될까?

```c
while (1)
{
    printf("계속 반복됩니다\n");
}
```

이 코드는
**절대 멈추지 않는다.**

이것을 **무한 루프**라고 한다.

무한 루프에 빠졌다면
실행 창을 닫거나
강제로 프로그램을 종료하자.

---

## 😵‍💫 초보자가 자주 하는 실수

* 반복 조건을 바꾸지 않음
* 증가식(`i++`)을 빼먹음
* 조건이 항상 참이 됨

문제가 생기면,
코드와 상황을 **그대로 GPT에게 보여 주자.**

---

> 초보자 기준으로 설명해 주세요.”
