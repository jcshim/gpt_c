# 2장. main 함수는 왜 꼭 필요할까?

1장에서 우리는 아주 중요한 경험을 했다.
C언어 코드를 작성했고, **실제로 실행했다.**

그런데 한 가지 이상한 점이 있다.
우리가 쓴 모든 C 코드에는 공통으로 등장하는 줄이 있다.

```c
int main(void)
```

왜 항상 이 줄이 필요할까?
이 질문을 혼자 고민하지 말고, **GPT에게 먼저 물어보자.**

---

## 💬 GPT에게 이렇게 물어보자

> **“C언어에서 main 함수는 왜 꼭 필요하나요?”**

GPT는 보통 이렇게 설명해 줄 것이다.

* 프로그램이 시작되는 위치다
* 컴퓨터는 어디서부터 실행해야 할지 알아야 한다
* main 함수가 프로그램의 출발점이다

여기서 핵심은 이것이다.

> **컴퓨터는 눈치가 없다.
> 그래서 ‘시작점’을 반드시 알려줘야 한다.**

그 시작점이 바로 `main` 함수다.

---

## 🚪 프로그램의 입구라고 생각하자

C 프로그램을 하나의 건물이라고 생각해 보자.

* 함수는 방이다
* 변수는 방 안의 물건이다

그렇다면 `main` 함수는 무엇일까?

> **프로그램의 정문, 입구다.**

컴퓨터는 프로그램을 실행할 때
항상 이렇게 묻는다.

> “어디서부터 시작하면 되지?”

그 질문에 대한 대답이 바로 `main`이다.

---

## ✋ main 함수가 없으면 어떻게 될까?

그럼 정말 `main`이 없으면 안 될까?
이것도 직접 확인해 보자.

아래처럼 코드를 바꿔 보자.

```c
#include <stdio.h>

void start(void)
{
    printf("Hello!\n");
}
```

그리고 실행해 보자.

결과는?

👉 **실행되지 않는다.**
👉 컴파일 에러가 발생한다.

컴퓨터는 이렇게 말하고 있는 것이다.

> “시작점(main)이 없다.
> 어디서 실행해야 할지 모르겠다.”

---

## 🔍 main 함수의 모양은 왜 이렇게 생겼을까?

다시 main 함수를 보자.

```c
int main(void)
```

지금은 이 줄을 **형태 그대로 받아들이면 충분하다.**

* `main` : 시작 함수의 이름
* `()` : 실행할 때 필요한 정보(지금은 없음)
* `{ }` : 실제 실행되는 코드 범위

`int`와 `return 0;`의 의미는
**나중에 다시 자세히 다룬다.**

지금은 이렇게만 기억하자.

> **main 함수 안에 있는 코드가
> 위에서 아래로 순서대로 실행된다.**

---

## ▶ main 함수 안에서 실행 순서 확인하기

아래 코드를 실행해 보자.

```c
#include <stdio.h>

int main(void)
{
    printf("첫 번째 줄\n");
    printf("두 번째 줄\n");
    printf("세 번째 줄\n");
    return 0;
}
```

실행 결과는 다음과 같다.

```
첫 번째 줄
두 번째 줄
세 번째 줄
```

이 결과를 통해 한 가지를 확인할 수 있다.

> **C언어는 main 함수 안에서
> 위에서 아래로 차례대로 실행된다.**

---

## 😵‍💫 초보자가 자주 겪는 에러

main 함수에서 가장 많이 하는 실수는 이것이다.

### ❌ 중괄호 `{ }`가 맞지 않는 경우

```c
int main(void)
    printf("Hello\n");
    return 0;
}
```

이 코드는 실행되지 않는다.
**중괄호는 ‘실행 범위’를 정하는 약속**이기 때문이다.

문제가 생기면,
에러 메시지를 그대로 GPT에게 보여 주자.

> 초보자 기준으로 설명해 주세요.”
