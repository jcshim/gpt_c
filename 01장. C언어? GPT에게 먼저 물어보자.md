## 1장. C언어? GPT에게 먼저 물어보자

처음 C언어를 배우는 사람에게 가장 어려운 것은 문법이 아니다.
**“무엇부터 해야 하는지 모른다”**는 점이다.

그래서 이 책은 C언어 설명부터 시작하지 않는다.
먼저 **GPT에게 질문하는 방법**부터 배운다.

---

### 1. C언어를 배우기 전에, 이렇게 물어보자

GPT에게 다음과 같이 질문해 보자.

> 💬 **GPT에게 질문**
> “C언어는 어떤 언어인가요? 초보자 눈높이로 설명해 주세요.”

GPT는 보통 다음과 같은 답을 준다.

* C언어는 컴퓨터와 가까운 언어이다.
* 속도가 빠르고, 운영체제·게임·임베디드 시스템에 많이 쓰인다.
* 다른 언어(C++, Java, Python)의 기초가 된다.

여기서 중요한 것은 **모두 이해하려 하지 않아도 된다는 점**이다.
지금은 단 한 가지만 기억하면 된다.

> **C언어는 ‘컴퓨터에게 일을 시키는 가장 기본적인 방법’이다.**

---

### 2. C언어로 할 수 있는 것들

C언어로 다음과 같은 것들을 만들 수 있다.

* 숫자 계산 프로그램
* 키보드 입력을 받는 프로그램
* 파일을 읽고 저장하는 프로그램
* 게임의 핵심 로직
* 그래픽 프로그램(raylib 사용)

지금은 “게임도 만들 수 있다” 정도만 기억하면 충분하다.

---

### 3. Visual Studio에서 C 프로그램 실행하기

이 책에서는 **Visual Studio 환경**을 그대로 사용한다.

* 새 프로젝트 생성
* 콘솔 애플리케이션 선택
* 언어: C

복잡한 설정은 필요 없다.
**코드를 쓰고 실행만 되면 성공이다.**

---

### 4. 첫 번째 C 프로그램

아래 코드를 그대로 입력해 보자.

```c
#include <stdio.h>

int main(void)
{
    printf("Hello, C!\n");
    return 0;
}
```

이제 실행한다.

화면에 다음 문장이 나오면 성공이다.

```
Hello, C!
```

이 순간이 **첫 번째 성공 경험**이다.

---

### 5. 이 코드에서 꼭 알아야 할 것 딱 3가지

지금은 모든 문법을 이해할 필요 없다.
아래 세 가지만 알면 된다.

1. `#include <stdio.h>`
   → 화면에 글자를 출력하기 위해 필요한 준비

2. `int main(void)`
   → 프로그램이 **여기서 시작된다**

3. `printf("Hello, C!\n");`
   → 화면에 글자를 출력한다

---

### 6. 이해가 안 되면, GPT에게 바로 묻자

이 코드가 이해되지 않는다면, 바로 GPT에게 묻는다.

> 💬 **GPT 질문 예시**
> “이 C 코드 한 줄씩 초등학생에게 설명해 주세요.”

또는

> 💬 “printf는 무슨 뜻인가요?”
> 💬 “main 함수는 왜 꼭 있어야 하나요?”

**이 책의 핵심 규칙은 하나다.**

> 👉 모르면 외우지 말고, GPT에게 묻는다.

---

### 7. 초보자가 가장 많이 만나는 에러

다음과 같은 실수를 자주 한다.

```c
printf("Hello, C!")
```

실행하면 에러가 난다.
이유는 **세미콜론(;)이 빠졌기 때문**이다.

```c
printf("Hello, C!");
```

C언어는 문장 끝에 `;`이 꼭 필요하다.
