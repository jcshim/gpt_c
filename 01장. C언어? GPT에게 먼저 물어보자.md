# 1장. C언어? GPT에게 먼저 물어보자

C언어를 배우기 전에, 우리는 질문부터 해야 한다.
이 책은 **설명으로 시작하지 않는다. 질문으로 시작한다.**

왜냐하면 지금은
👉 *혼자 외워서 배우는 시대가 아니라*
👉 **GPT와 대화하며 이해하는 시대**이기 때문이다.

---

## 💬 GPT에게 이렇게 물어보자

가장 먼저 GPT에게 다음 질문을 던져 보자.

> **“C언어는 뭐고, 왜 아직도 배우나요?”**

GPT는 아마 이런 이야기를 해줄 것이다.

* C언어는 빠르고 가볍다
* 운영체제, 임베디드, 게임 엔진의 기반 언어다
* C를 알면 다른 언어의 구조가 보인다

이 설명을 지금 모두 이해할 필요는 없다.
중요한 것은 단 하나다.

> **C언어는 오래된 언어가 아니라,
> 지금도 ‘현장에서 쓰이는 언어’라는 사실이다.**

---

## 🤔 그럼, 우리는 무엇을 할 것인가?

이제 방향은 분명해졌다.

* 문법을 외우는 대신
* 개념을 혼자 끙끙대는 대신

👉 **GPT에게 묻고, 직접 확인한다.**

그래서 이 장의 목표는 딱 하나다.

> **“C언어가 실제로 돌아간다는 것을 눈으로 확인하는 것”**

---

## 💻 도구는 필요하다 — Visual Studio

질문만으로는 부족하다.
확인하려면 **실행**이 필요하다.

이 책에서는 **Visual Studio**를 사용한다.

이유는 단순하다.

* 학교 실습실과 동일
* 대부분의 PC에서 바로 사용 가능
* 복잡한 설정 없이 실행 가능

지금은 도구의 원리를 이해할 필요가 없다.
**“C 코드를 실행하기 위한 버튼”**이라고 생각하면 충분하다.

---

## ▶ Visual Studio에서 첫 C 프로그램 실행하기

아래 단계는 **이해하지 말고 그대로 따라 하자.**

### 1️⃣ 새 프로젝트 만들기

* Visual Studio 실행
* **새 프로젝트 만들기** 클릭
* **빈 프로젝트** 선택
* 프로젝트 이름: `hello_c`

### 2️⃣ C 소스 파일 추가

* 솔루션 탐색기 → **소스 파일** 우클릭
* **추가 → 새 항목**
* 파일 이름: `main.c`

---

## ✨ 첫 번째 C 코드

아래 코드를 **그대로 복사해서 붙여 넣자.**

```c
#include <stdio.h>

int main(void)
{
    printf("Hello, C!\n");
    return 0;
}
```

이제 실행해 보자.

* **Ctrl + F5** 누르기
* 또는 **디버거 없이 시작**

검은 창이 뜨고 다음 문장이 보이면 성공이다.

```
Hello, C!
```

---

## 🎉 이 순간, 중요한 변화가 일어났다

아직 우리는 이런 것들을 모른다.

* `#include`가 뭔지
* `int main()`이 왜 필요한지
* `printf`의 원리

하지만 **전혀 상관없다.**

왜냐하면 이미 다음 사실이 증명되었기 때문이다.

> **C언어는 실제로 실행된다.
> 그리고 나는 직접 실행해 봤다.**

이 한 줄의 경험이
앞으로 모든 학습의 출발점이 된다.

---

## 😵‍💫 실행이 안 될 때, 이렇게 질문하자

만약 실행이 되지 않는다면
혼자 고민하지 말고 GPT에게 그대로 물어보자.

💬 **GPT 질문 예시**

> “Visual Studio에서 C언어로 Hello World를 실행하려고 합니다.
> 아래 코드가 실행되지 않는데,
> 완전 초보자 기준으로 하나씩 확인해 주세요.”

GPT는
👉 에러 메시지를 읽는 법
👉 가장 흔한 실수를
👉 순서대로 알려줄 것이다.
