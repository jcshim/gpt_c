# 5장. 선택의 힘 — if / else

지금까지 우리는  
- 숫자를 저장했고  
- 계산했고  
- 키보드로 값을 입력받았다  

이제 프로그램이 **선택**을 하게 만들어 보자.

사람은 항상 질문하며 산다.

> “맞아?”  
> “틀려?”  
> “이 경우야, 저 경우야?”

컴퓨터도 마찬가지다.  
그 선택을 담당하는 것이 **if / else**다.

---

## 💬 GPT에게 이렇게 물어보자

> **“if문을 사람의 말로 설명해 주세요.”**

GPT는 보통 이렇게 설명한다.

- 조건이 맞으면 이것을 하고  
- 아니면 다른 것을 한다  

아주 간단히 말하면 이렇다.

> **if는 ‘만약에’,  
> else는 ‘그렇지 않으면’이다.**

---

## 🔀 선택은 질문이다

if 문은 항상 **질문 하나**로 시작한다.

> “이 조건이 참인가?”

- 참이면 → if 안의 코드 실행  
- 거짓이면 → else 안의 코드 실행  

---

## ▶ 가장 기본적인 if 문

아래 코드를 실행해 보자.

```c
#include <stdio.h>

int main(void)
{
    int age;

    printf("나이를 입력하세요: ");
    scanf("%d", &age);

    if (age >= 20)
    {
        printf("성인입니다.\n");
    }
    else
    {
        printf("미성년자입니다.\n");
    }

    return 0;
}
````

실행 예시:

```
나이를 입력하세요: 17
미성년자입니다.
```

```
나이를 입력하세요: 25
성인입니다.
```

여기서 중요한 점은 이것이다.

> **조건이 바뀌면
> 결과도 바뀐다.**

---

## 🔍 조건식은 어떻게 생겼을까?

if 뒤에 나오는 부분을 자세히 보자.

```c
age >= 20
```

이것을 **조건식**이라고 한다.

자주 쓰는 조건 연산자는 다음과 같다.

| 기호   | 의미     |
| ---- | ------ |
| `>`  | 크다     |
| `<`  | 작다     |
| `>=` | 크거나 같다 |
| `<=` | 작거나 같다 |
| `==` | 같다     |
| `!=` | 같지 않다  |

조건식의 결과는 항상 둘 중 하나다.

> **참(true) 또는 거짓(false)**

---

## ⚠️ 가장 많이 틀리는 실수 하나

### ❌ `=` 와 `==` 를 헷갈리는 경우

```c
if (age = 20)
```

이 코드는 **비교가 아니다.**
값을 대입해 버린다.

비교할 때는 반드시 이렇게 써야 한다.

```c
if (age == 20)
```

---

## 💬 GPT에게 이렇게 물어보자

> **“C언어에서 = 와 == 는 뭐가 다른가요?”**

GPT는 이렇게 설명해 줄 것이다.

* `=`  : 값을 넣는다
* `==` : 값이 같은지 비교한다

이 차이를 헷갈리면
프로그램이 엉뚱하게 동작한다.

---

## 🧪 조건을 조금 더 바꿔 보자

아래 코드로 바꿔 실행해 보자.

```c
if (age < 8)
{
    printf("유아입니다.\n");
}
else if (age < 20)
{
    printf("학생입니다.\n");
}
else
{
    printf("성인입니다.\n");
}
```

이제 프로그램은
**세 가지 선택**을 할 수 있다.

---

## 😵‍💫 초보자가 자주 겪는 문제

* 조건에 괄호를 안 씀
* 중괄호 `{}` 위치 오류
* `==` 대신 `=` 사용

문제가 생기면,
에러 메시지와 코드를 **그대로 GPT에게 보여 주자.**
