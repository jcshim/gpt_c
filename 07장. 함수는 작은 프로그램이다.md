# 7장. 함수는 작은 프로그램이다

지금까지 우리는

* 순서대로 실행했고
* 선택했고
* 반복했다

그런데 코드를 조금만 길게 쓰다 보면
이런 생각이 든다.

> “이 코드를
> 또 써야 하나?”
>
> “복사해서 붙여 넣는 게 맞을까?”

이 질문에 대한 답이 바로 **함수**다.
함수는 **코드를 정리하는 가장 강력한 방법**이다.

---

## 💬 GPT에게 이렇게 물어보자

> **“함수는 왜 필요한가요?
> 안 쓰면 안 되나요?”**

GPT는 보통 이렇게 설명한다.

* 같은 일을 여러 번 쓰지 않기 위해
* 코드를 읽기 쉽게 만들기 위해
* 수정하기 쉽게 만들기 위해

한 문장으로 줄이면 이렇다.

> **함수는
> ‘자주 쓰는 코드를 이름 붙인 묶음’이다.**

---

## 🧩 함수는 작은 프로그램이다

함수를 이렇게 생각해 보자.

* 프로그램 전체가 큰 기계라면
* 함수는 그 안의 **작은 기계**다

작은 기계 하나가
하나의 일을 전담한다.

---

## ▶ 가장 단순한 함수 만들어 보기

아래 코드를 실행해 보자.

```c
#include <stdio.h>

void hello(void)
{
    printf("Hello, function!\n");
}

int main(void)
{
    hello();
    hello();
    return 0;
}
```

실행 결과는 다음과 같다.

```
Hello, function!
Hello, function!
```

여기서 중요한 사실은 이것이다.

> **같은 코드를
> 두 번 쓰지 않았다.**

---

## 🔍 함수의 기본 구조 살펴보기

```c
void hello(void)
{
    printf("Hello, function!\n");
}
```

지금은 이렇게 이해하면 충분하다.

* `hello` : 함수의 이름
* `{ }` : 함수가 하는 일
* `void` : 지금은 아무 값도 돌려주지 않음

그리고 함수는
**만들기만 하면 실행되지 않는다.**

---

## ▶ 함수는 반드시 호출해야 실행된다

아래 줄이 없다면 함수는 절대 실행되지 않는다.

```c
hello();
```

이 한 줄이 의미하는 것은 이것이다.

> **“hello라는 일을
> 지금 해 줘.”**

---

## ➕ 값을 주고 결과를 받는 함수

이번에는 숫자를 계산하는 함수를 만들어 보자.

```c
#include <stdio.h>

int add(int a, int b)
{
    return a + b;
}

int main(void)
{
    int result;

    result = add(3, 5);
    printf("결과: %d\n", result);

    return 0;
}
```

실행 결과는 다음과 같다.

```
결과: 8
```

여기서 핵심은 이것이다.

> **함수는
> 값을 받아서,
> 계산하고,
> 결과를 돌려줄 수 있다.**

## 🤔 return은 무엇을 할까?

이 질문도 GPT에게 던져 보자.

---

## 💬 GPT에게 이렇게 물어보자

> **“return은 왜 필요한가요?”**

GPT는 이렇게 설명해 줄 것이다.

* 함수의 결과를 돌려준다
* 함수의 실행을 끝낸다

즉,

> **return은
> ‘이게 결과야’라고 알려주는 신호다.**

* return 타입과 실제 값이 다름

이런 문제가 생기면
